# Health Check Module

## Описание
Модуль `Health Check` предоставляет простой эндпоинт для проверки работоспособности сервиса.
Используется балансировщиками нагрузки и системами мониторинга.

## API
Основные контроллеры:
-   `HealthCheckController` (`/api/v1/health-check`):
    -   `GET /`: Возвращает 200 OK, если сервис работает.

## Архитектура и реализация

### Основные компоненты
-   **HealthCheckController**: Единственный компонент. Возвращает статус 200 OK.

### Команды (Commands)
В данном модуле **не используется** паттерн Command.

### Назначение модуля
Модуль используется для проверки работоспособности сервиса внешними системами:
- Балансировщиками нагрузки
- Системами мониторинга
- CI/CD пайплайнами

### Основные потоки данных
1. **Проверка работоспособности**:
   - Внешняя система отправляет GET запрос на /api/v1/health-check
   - HealthCheckController возвращает статус 200 OK
   - Если запрос успешен, система считает сервис работающим

### Интеграции с другими модулями
- Нет прямых интеграций с другими модулями

### Важные особенности
- Модуль очень простой и не содержит сложной логики
- Используется только для проверки доступности сервиса
- Модуль использует Dependency Injection для подключения зависимостей
