# Модуль событий

Этот модуль предоставляет функциональность для управления событиями и доступом к ним.

## Компоненты

- `AdminEventsListPage` - Страница списка событий администратора
- `AdminEventsDetailsPage` - Страница деталей события администратора

## Возможности

- Создание и управление событиями
- Настройка доступа участников к событиям
- Просмотр деталей событий
- Управление участниками событий
- Запуск и остановка событий

## Технические детали

### Архитектура модуля

Модуль событий следует модульной архитектуре с четким разделением ответственности:
- `src/modules/events/` - Бизнес-логика и хуки для работы с событиями
- `src/pages/events/` - Компоненты страниц событий
- `src/shared/api/events/` - API клиент для взаимодействия с бэкендом
- `src/modules/events/widgets/` - Виджеты для отображения интерфейса событий

### API интеграция

Модуль использует `EventsApi` для взаимодействия с бэкендом:
- `getEvents` - Получение списка событий
- `getEventById` - Получение события по ID
- `createEvent` - Создание события
- `updateEvent` - Обновление события
- `deleteEvent` - Удаление события
- `getEventParticipants` - Получение участников события
- `addEventParticipants` - Добавление участников к событию
- `removeEventParticipants` - Удаление участников из события
- `startEvent` - Запуск события
- `stopEvent` - Остановка события

### Типы данных

Модуль использует строгую типизацию для всех данных событий:
- `IEvent` - Тип для события
- `IEventParticipant` - Тип для участника события
- `IEventAccess` - Тип для доступа к событию

## Зависимости

- Модуль событий (`src/modules/events`)
- Events API (`src/shared/api/events`)
- Модуль обзора (`src/modules/overview`)

## Использование

Модуль используется администраторами для создания и управления событиями в системе. Предоставляет интерфейс для настройки параметров событий и управления доступом участников.
