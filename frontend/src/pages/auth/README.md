# Модуль аутентификации

Этот модуль отвечает за всю функциональность, связанную с аутентификацией в приложении.

## Компоненты

- `AuthModulePage` - Главная точка входа в модуль аутентификации
- `AuthPage` - Основная страница аутентификации с формами входа/регистрации
- `ForgotPasswordPage` - Страница восстановления пароля
- `ResetPasswordPage` - Страница сброса пароля

## Возможности

- Аутентификация пользователя (вход/выход)
- Процесс восстановления пароля
- Интеграция социальной аутентификации
- Управление сессией

## Технические детали

### Архитектура модуля

Модуль аутентификации следует модульной архитектуре с четким разделением ответственности:
- `src/modules/account/` - Бизнес-логика и хуки для работы с аккаунтами
- `src/pages/auth/` - Компоненты страниц аутентификации
- `src/shared/api/auth/` - API клиент для взаимодействия с бэкендом

### Хуки данных

Модуль предоставляет следующие хуки для работы с аутентификацией:
- `useAuth` - Основной хук для управления аутентификацией
- `useAuthAnonymouslyLoader` - Хук для анонимной аутентификации
- `useCreateProfileLoader` - Хук для создания профиля
- `useGetAuthGate` - Хук для получения данных аутентификации
- `useGetAuthLoader` - Хук для загрузки данных аутентификации
- `useRequestPasswordRecoveryGate` - Хук для восстановления пароля
- `useUpdateAccountLoader` - Хук для обновления аккаунта
- `useGoogleAuth`, `useYandexAuth`, `useVkAuth`, `useMailRuAuth` - Хуки для социальной аутентификации

### API интеграция

Модуль использует `AuthApi` для взаимодействия с бэкендом:
- `login` - Вход в систему
- `logout` - Выход из системы
- `register` - Регистрация нового пользователя
- `forgotPassword` - Восстановление пароля
- `resetPassword` - Сброс пароля
- `googleAuth` - Аутентификация через Google
- `yandexAuth` - Аутентификация через Yandex
- `vkAuth` - Аутентификация через VK
- `mailRuAuth` - Аутентификация через Mail.ru

### Типы данных

Модуль использует строгую типизацию для всех данных аутентификации:
- `IUser` - Тип для пользователя
- `IAuthResponse` - Тип для ответа аутентификации
- `IRegisterRequest` - Тип для запроса регистрации
- `IForgotPasswordRequest` - Тип для запроса восстановления пароля
- `IResetPasswordRequest` - Тип для запроса сброса пароля

## Зависимости

- Модуль аккаунта (`src/modules/account`)
- Auth API (`src/shared/api/auth`)
- React Query для управления состоянием данных
- `@react-oauth/google` для Google аутентификации

## Использование

Модуль аутентификации обычно доступен через основной роутер приложения и предоставляет защищенные маршруты для аутентифицированных пользователей.
