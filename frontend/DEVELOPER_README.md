## Архитектура проекта

Подробная информация об архитектуре проекта находится в файле [architecture.md](architecture.md).

Основные принципы архитектуры:
- Модульная архитектура с четким разделением ответственности
- Использование React Query для управления серверным состоянием
- Использование Effector для управления клиентским состоянием (deprecated)
- Следование принципам Feature-Sliced Design

## Разработка и развертывание

### Локальная разработка
1. Убедитесь, что у вас установлен Node.js >= 22 и Yarn >= 1.22
2. Выполните `nvm use` для установки правильной версии Node.js
3. Создайте файл .env с необходимыми переменными окружения
4. Установите зависимости: `yarn install`
5. Запустите приложение: `yarn start`

### Основные команды
- `yarn start` - Запуск приложения в режиме разработки
- `yarn build` - Сборка приложения для продакшена
- `yarn storybook` - Запуск Storybook для разработки компонентов
- `yarn lint` - Проверка кода с помощью ESLint
- `yarn format` - Форматирование кода с помощью Prettier

### CI/CD процессы
- `.github/workflows/build.yml` - Сборка и тестирование
- `.github/workflows/deploy-prod.yml` - Развертывание в продакшн
- `.github/workflows/deploy-test.yml` - Развертывание в тестовое окружение
- `.github/workflows/publish-storybook.yml` - Публикация Storybook

### Соглашения
- Соглашения по именованию веток: `feature/short-description`
- Именования коммитов: conventional commits
- Строгая типизация во всем проекте
- Использование ESLint и Prettier для контроля качества кода
